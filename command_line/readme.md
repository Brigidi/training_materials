# shell
a "shell" is the window where you enter commands that directly interact with a computer's operating system. please note: this repo is largely written from the perspective of using terminal with mac/linux. the experience might differ for people using other machines/operating systems.

## where you will use shell
anytime you use a computer, you have an opportunity to use shell syntax.

in most cases, people will engage with a computer's operating system using graphical-user interfaces (also known as "gui", which is pronounced "gooey"). this provides a point-and-click experience by using applications that translate the things you want to do to the computer; rather than making you submit a sequence of text-based commands.

## when you will use shell
this is not an exhaustive list and is merely coming from my experience. you can use shell as much as you want — and i've been attempting to do so because i find it more liberating than point-and-click experiences.

with that said, here's a breif overview of some instances that i think will be most common:

- programming with python 
- managing files
- working with servers
- computer networking 

## how you will use shell
it depends on the circumstances but all of my experiences using shell come via the **terminal**. so, if you use a mac or linux computer, then your machine will have terminal already available. i have not used a windows computer in 20 years, so i'm not as confident in what they use but i _think_ it's common for **power shell** to be the native shell.

additionally, when using an **integrated development environment** (aka "ide") such as pycharm or vscode, they will have their own **terminal** interface that will allow you to use command-line syntax to manage your project and interact with your machine. 

in these situations, instances where you are working in a development environment, you might want to use the terminal to:

1. create new **directories** (aka folders)
2. create new **files** 
3. organize your files and directories
4. copy files from existing directories
5. manage version control using **git**
6. install packages or libraries you need for your project (aka "programs", "dependencies", and/or "requirements)

## why you will use shell
generally speaking there are many reasons, so here are some that come to mind:

1. it's a more liberated computing experience when attempting to do sequential actions
2. you are programming something
3. you are using an ai tool like claude, which processes commands using command-line and markdown

of course, that's a broad generalization; if you're reading this an have other ideas please reach out and/or update the list.

## shell languages
there are variations of shell syntax, which means some of the commands listed in this repo might not 1:1 with language used by your machine and/or environment. if that's the case, you can use web-search and/generative ai to identify what is applicable to you.

## how to identify your shell language
open up your terminal interface and enter the following command:

```echo $SHELL```
