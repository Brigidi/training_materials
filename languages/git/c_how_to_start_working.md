when working in a git repo, it's wise to begin each session with the following commands:

```shell

git branch

```

this will ensure that you are working on main instead of potentially continuing to work on a development branch.

```shell

git pull

```

this will ensure your repo is up-to-date with any changes that might have been pushed since you last worked.

# why you should do this
it ensures that you do not develop a divergent logic set that varies from the most recent contributions to the project
