*git* is an open-source version control system that tracks changes in projects, which facilitates collaboration when composing computer logic.

## where you will use git
most likely within an Integrated Development Environment (aka 'IDE') via the [shell](/languages/shell) terminal.

## when you will use git
anytime you are writing computer code.

## how you will use git
you will create a project directory (aka 'folder') and you will initialize git. this will technically change your directory into a repository, because you can now track changes. you will add files to your repo and commit all your changes. you will *likely* store your repo on github, which will allow others to access your code; it will also allow you to access your code regardless of the machine you are using.

## why you will use git
you will have the ability to contribute to code in a central location, which means you can change code over time. Additionally, it will allow you to revert code if needed; to collaborate with others; and, most importantly, it will minimize the need to manually transfer code.
